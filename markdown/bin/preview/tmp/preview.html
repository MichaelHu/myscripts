如何提交和维护自己的文档

一、文档编写及存放

1.1 文档后缀

文档使用markdown语法编写，建议保存成.md后缀的文本文件，文件编码统一为UTF-8。

1.2 文档保存和WEB访问

文档保存在svn服务器上，每个同学有一个用自己名字命名的子目录，在这个子目录下，你可以随意存放你的文档。

  1. svn服务器地址：http://db-news-fe2.vm.baidu.com:8520/svn/newsdocs
  2. 文档WEB访问地址： http://db-news-fe2.vm.baidu.com:8520/docs/USERNAME/DIR/TO/DOC/FILE.html

二、创建、编写和提交文档

以下举例说明，如何创建、编写以及提交文档：

  1. checkout出属于自己的文档根目录至本地，以hudamin为例：
    svn co http://db-news-fe2.vm.baidu.com:8520/svn/newsdocs/hudamin
  2. 进入根目录，建立并进入子目录
    cd hudamin
mkdir test
cd test
  3. 添加新文档
    > vim test-howto.md
  4. 编辑内容，在文件中添加以下内容：
    # 这里是文章大标题
这里是引言
## 一、第一部分
第一部分内容
    code
    $('body').css('background', '#000');
## 二、第二部分
* 无序列表
* 无序列表
    1. 嵌套有序列表
            inner code
            console.log('hello');
    2. 嵌套有序列表
  5. 退出，预览效果：
    markdown_preview test-howto.md

    预期的效果应该如下所示：

  6. 如不符合预期，再行修改。如果符合预期，进入下一步。
  7. 提交文档：
    svn add test-howto.md
svn ci -m "initial commit: add test-howto.md, by hudamin"
  8. WEB访问，在浏览器中键入

    http://db-news-fe2.vm.baidu.com:8520/docs/hudamin/test/test-howto.md.html ，不出意外的话，可以看到第5点看到的效果图。

三、一些约定

  1. 每个同学尽量在自己的目录下进行操作，避免多人同时修改，造成版本冲突
  2. 提交文档时，必须带操作说明，说明格式如下：
    initial commit, by hudamin
    • 格式为：操作内容的说明，接上提交人
    • 尽可能使用英文
  3. 文档名称、目录名称不要使用中文，也不要带空格

四、本地预览

为了避免必须通过svn提交以后才能查看最终效果，提高编写代码的效率， 可以通过下载以下工具包，并在本地部署好。就可以通过预览功能，先预览无误以后，再行提交。

  • 下载地址：markdown_preview_tar.gz[2014-08-07更新版]
  • 部署以及使用方法：
    1. Windows：
      1. 下载以后解包至C:\markdown_preview
      2. 将C:\markdown_preview\markdown.bat拷贝至C:\WINDOWS\system32
      3. 使用方式：
        markdown.bat FILENAME
      4. 按以上方式解包，即可正常使用。如果需要自己更换目录，则需要修改markdown.bat中对应路径参数。

        如果不熟悉bat文件，则不建议修改，而使用默认部署

    2. Mac:
      1. 下载以后解包至某一本地目录，例如DIR/markdown_preview
      2. 修改DIR/markdown_previw/markdown.sh，将
        MARKDOWNCMD=/Users/hudamin/projects/git/git-myscripts/markdown/bin/preview/markdown.out
ROOT=/Users/hudamin/projects/git/git-myscripts/markdown/bin/preview

        修改成：

        MARKDOWNCMD=DIR/markdown_preview/markdown.out
ROOT=DIR/markdown_preview
      3. 将DIR/markdown_preview/markdown.sh拷贝成~/bin/markdown
      4. 设置可执行权限
        chmod +x markdown
      5. 使用方式：
        markdown FILENAME
